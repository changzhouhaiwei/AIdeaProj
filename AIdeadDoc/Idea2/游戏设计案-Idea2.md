# 《方块填充消除》游戏设计案

> 依据 `Idea2` 草稿整理，版本与 Idea2 同步迭代。

## 一、定位与对标

- **品类**：休闲益智 / 方块放置消除。
- **规则气质**：整体接近「俄罗斯方块式」的**整行/整列（或满格区域）消除**思路；体验与市面热门 **Block Blast** 类产品相近——**无重力下落**，以**拖拽放置**为主。
- **一句话**：在固定棋盘上，用下方随机给出的若干块形拼入棋盘，填满行列（或规则指定的区域）即消除，持续得分/闯关。

## 二、核心玩法循环

1. **观察**：上方为固定大小的可放置区域（当前设定 **8×8**）。
2. **决策**：下方每次给出 **3 个随机形状**（块组），玩家选择其一。
3. **操作**：**拖动**形状到上方区域；**仅当所有占用格均在棋盘内且不与已占格重叠**时，才可成功放下。
4. **结算**：放置后触发**消除判定**（与 MinCell 粒度一致）；消除后腾出空间。
5. **续局**：用尽或无法放置时进入失败/结算；或按关卡目标进入下一关（若后续扩展）。

## 三、棋盘与操作规则

| 项目 | 设定 |
|------|------|
| 棋盘 | 8×8 |
| 每回合候选块 | 3 个随机形状 |
| 放置方式 | 拖拽至棋盘区域，位置合法则填充 |
| 最小逻辑单元 | `MinCell`：占用、碰撞、消除等均基于单元格 |

### 待补设计（建议后续定稿）

- **消除条件**：仅满行/满列，还是包含 2×2 / 区域等变体。
- **无解时**：直接结束、刷新一批形状、或道具救济。
- **计分与关卡**：分数规则、连击、关卡目标（若有）。

## 四、形状库

### 矩形条块

- 直线：`1×1`、`1×2`、`1×3`、`1×4`、`1×5`
- 其它矩形：`2×2`、`2×1`、`2×3`
- `3×2`（Idea2 原文中连续两行均为 `3×2`，**需确认是否其一应为 `3×1` 或 `3×3` 等**）
- `4×1`、`5×1`

### 多连块（四方向旋转）

- **Z 字形**：上 / 下 / 左 / 右四种朝向。
- **L 字形**：上 / 下 / 左 / 右；规模 **2** 与 **3**（两种 L 尺寸）。
- **大 L**：上 / 下 / 左 / 右；**3×3** 尺度的大 L 轮廓。

### 实现说明

所有形状在 `ShapeDefine` 中定义为相对于锚点的占用格集合；`ShapeNode` 实例化时映射到多个 `MinCell`。

## 五、程序结构（与工程命名对齐）

| 模块 | 职责 |
|------|------|
| `BlockGameManager` | 整局流程：回合、胜负、分数、与 UI/关卡衔接 |
| `BoardMgrNode` | 棋盘格状态、合法落点检测、放置与消除后的刷新 |
| `WaitNode` | 下方 3 个候选形状的生成、展示与交互入口 |
| `ShapeNode` | 单个可拖拽形状；由形状数据驱动子级 `MinCell` 布局 |
| `ShapeDefine` | 全部形状的定义（含旋转集合） |
| `MinCell` | 最小格子实体/数据；消除与占用的基础单位 |

## 六、扩展方向（可选）

- **关卡**：步数限制、指定消除次数、障碍物格。
- **外围**：金币、皮肤、每日挑战。
- **难度**：形状池权重、棋盘尺寸、每回合候选块数量变化。
